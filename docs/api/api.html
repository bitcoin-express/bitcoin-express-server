patch /settings

Old: /setConfig

Modifies chosen account's settings

Secured by passthrough

By default each request - unless stated otherwise - requires authentication header passed together with the request.

  • Request
  • Security

Body

Media type: application/json

Type: object

Properties
  • default_payment_currency: (XBT)

    Default currency for transactions. It will be used by default if there is no explicit xurrency set for a transaction.

  • default_payment_timeout: (string - default: 3600)

    Time in seconds after which a not-finished transaction will be marked as expired.

  • email_customer_contact: (string)

    Merchant's contact email for customer related queries

  • email_account_contact: (string)

    Merchant's contact email for account related queries

  • provide_receipt_via_email: (boolean - default: false)

    Is a merchant providing an option to send a receipt via email

  • provide_refund_via_email: (boolean - default: false)

    Is a merchant providing an option to send a refund via email

  • home_issuer: (string - default: be.ap.rmp.net)

    A default issuer to work with

  • acceptable_issuers: (array of )

    Issuers that the Merchant accepts Coins from

Secured by passthrough

Headers

  • BE-MG-Auth-Token: required (string)

post /accounts

  • Request
  • Response

Body

Media type: application/json

Type: object

Properties
  • domain: required (string)

    The domain name of the Merchant's service

  • email_account_contact: (string)

    Email address for account related queries

  • email_customer_contact: (string)

    Email address for customer related queries

  • name: (string)

    Name of the account's holder

Example:

{
  "domain": "mysite.url",
  "email_account_contact": "account@mysite.url",
  "email_customer_contact": "sales@mysite.url",
  "name": "John Smith"
}
